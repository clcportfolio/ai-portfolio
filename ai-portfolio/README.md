# AI Portfolio — Cody Culver

A portfolio of agentic AI projects built with Claude Code. Each project is a runnable,
demoable multi-agent pipeline targeting healthcare and life sciences workflows.

Built to demonstrate: multi-agent orchestration, structured LLM output, production
observability (Langfuse), guardrails and PHI awareness, and Streamlit deployment.

---

## How this is built

Every project in this portfolio is generated by a custom agentic build system living in
`ai-agents/`. Rather than writing projects by hand, a pipeline of specialized agents
constructs each one end-to-end:

```
orchestrator
    │
    ├─► software_engineer   — writes pipeline.py, agents/, app.py, requirements.txt
    ├─► evaluator           — scores the output (0–10), returns structured feedback
    │      └─ if "revise" and iterations < 3 → software_engineer loops with feedback
    │
    ├─► guardrail_engineer  — writes guardrails.py (PHI stub, injection scan, rate limiting)
    ├─► security_specialist — reviews for secrets, injection vectors, HIPAA risks
    └─► tech_writer         — writes nontechnical overview, technical overview, build walkthrough
```

Each build agent has its own `agent.py`, `prompts.py`, and `SKILLS.md`. The orchestrator
is the only flat file — it's the controller, not a content producer.

## Projects

### [Clinical Intake Router](ai-projects/projects/clinical-intake-router/)
Routes clinical intake forms to the right department at the right urgency level.
Three-agent pipeline (extraction → classification → routing) with RBAC, NL2SQL chatbot,
S3 + Supabase storage, async LLM calls, and a 4-layer guardrail on natural-language queries.

### [Clinical Trial Eligibility Screener](ai-projects/projects/clinical-trial-eligibility-screener/)
Evaluates a patient summary against trial inclusion/exclusion criteria and returns a
plain-English eligibility verdict (Eligible / Likely Ineligible / Needs Review) with
per-criterion reasoning. Three-agent pipeline: criteria extraction → per-criterion
evaluation → verdict synthesis.

### [Stereogram Converter](ai-projects/projects/stereogram-converter/)
Converts a greyscale depth map into a Magic Eye stereogram. No LLM — pure algorithm
using the SIRTS method (NumPy + Pillow). Demonstrates the apps/ building-block pattern.

## Structure

```
ai-projects/
├── projects/         ← Demoable projects (LLM pipelines + Streamlit UI)
├── apps/             ← Pure algorithms, no LLM (building blocks for projects)
├── ai-agents/        ← Build pipeline agents (orchestrator, software_engineer, etc.)
├── CLAUDE.md         ← Project constitution — conventions, patterns, hard rules
├── SKILLS.md         ← Reusable LangChain patterns and snippets
└── CHAT_LOG.md       ← Session log of all builds and decisions
```

Each project under `projects/` is independently runnable:
```bash
cd ai-projects/projects/<project-name>
pip install -r requirements.txt
cp .env.example .env   # add API keys
streamlit run app.py
```
